# μJava Compiler

A MicroJava compiler implementation written in Java, developed as a university project. This compiler translates MicroJava source code into bytecode that can be executed on the MicroJava Virtual Machine.

## About MicroJava

MicroJava is a simplified, Java-like programming language designed for educational purposes. It features a subset of Java's syntax with some unique extensions, making it ideal for learning compiler construction principles.

## Features

This compiler implements the complete MicroJava specification with several extended features:

### Language Extensions
- **Enums**: Type-safe enumeration types with automatic or explicit value assignment
- **Ternary Operator**: Conditional expressions (`condition ? expr1 : expr2`)
- **For Loops**: C-style for loops with initialization, condition, and iteration statements
- **Switch Statements**: Multi-way branching with case labels and break statements
- **Boolean Type**: Native `bool` type with `true`/`false` literals
- **Array Length**: `.length` property for arrays

### Core Features
- Constants, variables, and arrays
- Functions and methods with parameters
- Arithmetic expressions (`+`, `-`, `*`, `/`, `%`)
- Relational operators (`==`, `!=`, `<`, `>`, `<=`, `>=`)
- Logical operators (`&&`, `||`)
- Control flow (`if/else`, `while`, `for`, `switch/case`, `break`, `continue`)
- Built-in functions: `chr()`, `ord()`, `len()`
- I/O operations: `print()`, `read()`

## Compiler Phases

The compiler is implemented in four phases:

1. **Lexical Analysis** - JFlex-based lexer tokenizes the source code
2. **Syntax Analysis** - CUP parser generates an Abstract Syntax Tree (AST)
3. **Semantic Analysis** - Type checking, scope resolution, and semantic validation
4. **Code Generation** - MicroJava bytecode emission

## Requirements

- **Java Development Kit (JDK)** 8 or higher
- **Eclipse IDE** (recommended) with Java development tools
- **Apache Ant** (for regenerating lexer/parser from specifications)

## Setup and Usage (Eclipse)

### 1. Import the Project

1. Open Eclipse IDE
2. Go to `File` → `Import` → `Existing Projects into Workspace`
3. Select the `MJCompiler` directory
4. Click `Finish`

### 2. Build the Project

The project includes an Ant build file that regenerates the lexer and parser:

1. Right-click on `build.xml` in the Package Explorer
2. Select `Run As` → `Ant Build`
3. Choose the `compile` target to rebuild everything

**Note**: The `.classpath` file is configured for Eclipse. All dependencies (`lib/*.jar`) should be automatically recognized.

### 3. Compile a MicroJava Program

1. Place your MicroJava source file at `test/program.mj`
2. Run the `Compiler.java` class:
   - Right-click `test/rs/ac/bg/etf/pp1/Compiler.java`
   - Select `Run As` → `Java Application`
3. The compiler will generate `test/program.obj` (bytecode file)
4. Compilation logs are written to `logs/` directory

### 4. Run the Generated Bytecode

To execute the compiled program:

1. Right-click on `build.xml`
2. Select `Run As` → `Ant Build`
3. Choose the `runObj` target

Or use the `debugObj` target for debug mode with detailed execution trace.

## Example MicroJava Program

```java
program example

const int MAX = 100;

int count;

    {

        count = 0;
```

## Compiler Output

The compiler produces:
- **Abstract Syntax Tree (AST)** - Logged during parsing
- **Symbol Table Dump** - Shows all declared symbols with types and addresses
- **Bytecode File** (`.obj`) - Executable on the MicroJava VM
- **Compilation Logs** - Detailed logs in `logs/` directory

## Architecture Overview

### Symbol Table
- Uses `rs.etf.pp1.symboltable.*` library
- Pre-defined universe scope with `int`, `char`, built-in functions
- Supports nested scopes for methods and blocks

### AST Traversal
- AST nodes generated automatically by CUP with `-buildtree` flag
- Visitor pattern for semantic analysis and code generation
- Bottom-up traversal using `traverseBottomUp()`

### Code Generation
- Emits bytecode for the MicroJava Virtual Machine
- Stack-based architecture
- Built-in runtime functions for I/O and type conversions

## Development

To modify the compiler:

1. **Lexer**: Edit `spec/mjlexer.flex` and regenerate with JFlex
2. **Parser**: Edit `spec/mjparser_astbuild.cup` and regenerate with CUP
3. **Semantic Analysis**: Modify `src/rs/ac/bg/etf/pp1/SemanticAnalyzer.java`
4. **Code Generation**: Modify `src/rs/ac/bg/etf/pp1/CodeGenerator.java`

Run the `compile` Ant target after making changes to regenerate all generated files.

## Troubleshooting

### Common Issues

**Problem**: Eclipse doesn't recognize `mj-runtime` classes
**Solution**: The `.classpath` file should include all required JARs. If issues persist, manually add JARs from `lib/` to the build path.

**Problem**: "Class not found" errors when running
**Solution**: Make sure to run the `compile` Ant target first to generate all parser/lexer files.

**Problem**: Parser conflicts or grammar errors
**Solution**: Check the CUP output in the Ant console. The grammar must be LALR(1) compatible.
